# TransitOps — Fleet Operations & Smart Dispatch Dashboard

TransitOps is a modern, responsive, and feature-rich SaaS Fleet Management and Smart Dispatch Dashboard built with **Django**, **SQLite**, and premium **Vanilla CSS/HTML/JS** (featuring glassmorphism, CSS transition layers, and micro-interactions).

---

## 🌟 Key Features

### 1. Unified KPIs & Analytics Dashboard
* **Dynamic Fleet Insights:** Monitor Fleet Utilization, Active/Available Fleet counts, and Drivers on duty at a glance.
* **Financial Charts & Trends:** Integrates **Chart.js** displaying total revenue flows, breakdown metrics, and a line chart of daily revenue trends. Filters dashboard view by vehicle type dynamically.

### 2. Intelligent Trip Lifecycle & Mapping
* **Route Geocoding:** Select trip source and destination directly on an interactive map powered by **Leaflet.js** and **OpenStreetMap (Nominatim)**.
* **Smart Estimations:** Automatically calculates planned distance and estimates fuel costs in real-time based on selected fuel type (Diesel: ₹96/L, Petrol: ₹104/L, CNG: ₹76/L) assuming a baseline efficiency of 10 km/L.
* **State Operations:** Transitions trips through `Draft` -> `Dispatched` -> `Completed` / `Cancelled` stages.

### 3. Driver Safety & Points Deductions (Safety Officer Panel)
* **Interactive Point Deduction:** Safety Officers can deduct points directly from the **Trip Detail View** of their assigned trips via a premium modal. Every deduction logs the reason, point count, reporting officer, and timestamp.
* **Safety score recovery:** Drivers starting with less than 100 safety points automatically recover **+1 point per completed trip** until they reach 100.
* **Infraction Logs:** Infractions are fully tracked in the database and visible in the driver profile view.

### 4. Automated Fuel Logging & Financial Analysis
* **Failsafe Backend Estimators:** Calculates costs on the Python server should browser JavaScript be blocked.
* **Auto-Generated Logs:** Saves actual fuel logs based on drivers' real-time fuel consumed inputs using the specific fuel type pricing.
* **Total Operational Cost:** Aggregates fuel, maintenance, toll, and miscellanous costs dynamically on the redesigned **Fuel & Expenses** view.

### 5. Security & Rate Limiting
* **Failed Login Lockout (Rate Limiting):** Implements `LoginLockoutMiddleware` to temporarily block a client's IP address for 5 minutes after 5 consecutive failed login attempts, preventing brute-force authentication attacks.
* **Concurrent Session Limitation:** Enforces active session controls to restrict multiple concurrent logins under the same user account from different devices or sessions at the same time to protect user privacy and dispatch audit integrity.

---

## 🛠️ Technology Stack

* **Backend:** Python 3.x, Django 5.x / 6.x
* **Database:** SQLite
* **Frontend:** Vanilla HTML5, CSS3 (Modern HSL colors, Glassmorphism, CSS keyframe loading animations, Flex/Grid layouts), JavaScript (ES6+)
* **Libraries:** Chart.js (Charts), Leaflet.js (Interactive mapping)

## 🚀 Setup & Installation

### 1. Clone the Repository
```bash
git clone <repository_url>
cd Odoo-Hackathon-26
```

### 2. Configure Virtual Environment & Install Dependencies
Ensure you have Python installed, then set up the environment:
```bash
# Windows
python -m venv venv
venv\Scripts\activate

# Install Django
pip install django
```

### 3. Database Setup & Migrations
```bash
python manage.py makemigrations
python manage.py migrate
```

### 4. Load Mock/Demo Data
Pre-populate the system with vehicles, drivers, past fuel logs, maintenance metrics, and admin users:
```bash
python manage.py create_demo_data
```

### 5. Start the Development Server
```bash
python manage.py runserver
```
Visit the system locally at `http://127.0.0.1:8000/`.
